# Contract Sync - 기능 명세서

> AI 기반 외주용역 계약서 일정 추출 및 업무 관리 플랫폼

---

## 1. 시스템 개요

### 1.1 서비스 목적
외주용역 계약서를 업로드하면 AI(Google Gemini)가 추진 일정을 자동 추출하고, 업무 목록을 생성하여 대시보드에서 관리할 수 있는 웹 애플리케이션이다. 계약서에서 수동으로 일정을 정리하는 반복 업무를 자동화하여 프로젝트 관리 효율을 높이는 것이 핵심 가치이다.

### 1.2 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 백엔드 | FastAPI (Python 3.12) | 비동기 REST API 서버 |
| 프론트엔드 | HTML5 + Tailwind CSS + Alpine.js 3.x | SPA (Single Page Application) |
| 데이터베이스 | SQLite (aiosqlite, SQLAlchemy 비동기) | 로컬 데이터 저장 |
| AI 엔진 | Google Gemini 2.0 Flash (멀티모달) | 계약서 텍스트/이미지 분석 |
| 인증 | 이메일 인증 + Google OAuth 2.0 | 사용자 인증 |
| 파일 처리 | PyMuPDF, python-docx, olefile, Pillow | 다양한 문서 포맷 파싱 |
| 웹폰트 | Noto Sans KR | 한글 UI 렌더링 |

### 1.3 전체 사용자 흐름

```
[비로그인] 랜딩 페이지 → 회원가입/로그인
    ↓
[로그인 후] 대시보드 (업무 현황)
    ├── 계약서 등록 → 파일 업로드 → AI 분석 → 결과 확인/편집 → 저장
    ├── 업무 관리 → 상태 변경 / 담당자 지정 / 메모 작성
    ├── 마이페이지 → 내 계약 목록 조회/수정/삭제
    ├── 팀 관리 → 팀 생성 / 멤버 초대 / 역할 관리
    └── 설정 → 다크 모드 / 알림 확인
```

## 2. 사용자 인증

### 2.1 이메일 회원가입

**사용자 흐름:**

1. **이메일 입력 단계**
   - 사용자가 회원가입 버튼 클릭 → 로그인 모달이 열림
   - 이메일 주소를 입력하고 "인증코드 발송" 버튼 클릭
   - 서버에서 6자리 숫자 인증코드를 생성하여 SMTP 메일 발송
   - 인증코드 유효 시간: 10분
   - 이미 가입된 이메일인 경우 오류 메시지 표시

2. **인증코드 검증 단계**
   - 화면이 인증코드 입력 폼으로 전환됨
   - 6자리 숫자를 입력하면 자동으로 검증 요청 전송
   - 성공 시 비밀번호 설정 화면으로 전환
   - 실패 시 "잘못된 인증코드입니다" 오류 표시
   - "인증코드 재발송" 링크로 새 코드 요청 가능

3. **비밀번호 설정 단계**
   - 비밀번호 입력 필드 + 확인 필드
   - **비밀번호 강도 미터**: 실시간으로 강도를 시각 표시
     - 약함 (빨간색): 기본 조건만 충족
     - 보통 (노란색): 대소문자 + 숫자 혼합
     - 강함 (초록색): 12자 이상 + 특수문자 포함
   - 비밀번호 표시/숨기기 토글 (눈 아이콘)
   - 비밀번호 규칙: 8자 이상, 영문 + 숫자 필수
   - 가입 완료 시 자동 로그인 후 대시보드로 이동

**비밀번호 강도 점수 산정:**
| 조건 | 점수 |
|------|------|
| 8자 이상 | +1 |
| 12자 이상 | +1 |
| 대문자 포함 | +1 |
| 소문자 포함 | +1 |
| 숫자 포함 | +1 |
| 특수문자 포함 | +1 |

- 0~2점: 약함 (빨간색 바)
- 3~4점: 보통 (노란색 바)
- 5~6점: 강함 (초록색 바)

### 2.2 이메일 로그인

**사용자 흐름:**
1. 이메일 + 비밀번호 입력
2. 비밀번호 표시/숨기기 토글 지원
3. 로그인 성공 시 세션 쿠키 설정 후 대시보드 이동
4. 실패 시 "이메일 또는 비밀번호가 올바르지 않습니다" 표시

**보안 처리:**
- bcrypt 솔트 기반 해싱으로 비밀번호 저장
- 사용자가 존재하지 않는 경우에도 더미 해시를 검증하여 타이밍 공격 방지
- 세션 토큰: HttpOnly 쿠키, 24시간 유효, SameSite=Lax

### 2.3 Google OAuth 로그인

**사용자 흐름:**
1. "Google로 로그인" 버튼 클릭
2. Google 인증 페이지로 리다이렉트 (openid, email, profile 범위)
3. Google 인증 완료 후 콜백 URL로 리다이렉트
4. 기존 사용자: 세션 생성 후 대시보드 이동
5. 신규 사용자: 자동 가입 (Google 이름/프로필 사진 저장) 후 대시보드 이동

### 2.4 로그아웃
- 헤더의 프로필 아이콘 드롭다운에서 "로그아웃" 클릭
- 서버 세션 삭제 + 쿠키 제거
- 랜딩 페이지로 이동

### 2.5 세션 관리
| 항목 | 상세 |
|------|------|
| 저장 방식 | DB (user_sessions 테이블) |
| 토큰 형식 | URL-safe 32바이트 랜덤 문자열 |
| 유효 기간 | 24시간 |
| 쿠키 속성 | HttpOnly, SameSite=Lax, Secure(프로덕션) |
| 만료 세션 정리 | 1시간 주기 자동 삭제 |

### 2.6 API 엔드포인트

| 메서드 | 경로 | 설명 | Rate Limit |
|--------|------|------|------------|
| POST | `/api/v1/auth/send-code` | 인증코드 발송 | 5회/분 |
| POST | `/api/v1/auth/verify-code` | 인증코드 검증 | 5회/분 |
| POST | `/api/v1/auth/signup` | 회원가입 | 3회/분 |
| POST | `/api/v1/auth/login/email` | 이메일 로그인 | 3회/분 |
| GET | `/api/v1/auth/login/google` | Google 로그인 시작 | - |
| GET | `/api/v1/auth/callback/google` | Google 콜백 처리 | - |
| GET | `/api/v1/auth/me` | 현재 사용자 조회 (팀 목록 포함) | - |
| POST | `/api/v1/auth/logout` | 로그아웃 | - |

## 3. 계약서 분석 (AI 추출)

### 3.1 파일 업로드 화면

**UI 구성:**
- 화면 중앙에 드래그 앤 드롭 영역 (점선 테두리, 파일 아이콘)
- "파일을 끌어다 놓거나 클릭하여 선택" 안내 문구
- 지원 형식 표시: "PDF, DOCX, HWP, JPG, PNG (최대 50MB)"
- 파일 선택 시 파일명과 크기 표시
- "일정 추출 시작" 버튼 활성화

**드래그 앤 드롭 동작:**
- 파일을 드래그하여 영역 위에 올리면 배경색이 변경 (시각적 피드백)
- 드롭 시 파일 검증 후 자동 설정
- 클릭으로도 파일 탐색기를 통해 선택 가능

**파일 검증:**
| 항목 | 제한 | 오류 메시지 |
|------|------|-----------|
| 파일 크기 | 최대 50MB | "파일 크기는 50MB를 초과할 수 없습니다" |
| 확장자 | pdf, docx, doc, hwp, hwpx, jpg, jpeg, png, tiff, tif, bmp, webp | "지원하지 않는 파일 형식입니다" |
| Magic bytes | 바이너리 시그니처 검증 | "파일이 손상되었거나 형식이 올바르지 않습니다" |

### 3.2 지원 파일 형식 및 처리 방식

| 형식 | 확장자 | 처리 방식 | 파서 |
|------|--------|---------|------|
| PDF (텍스트) | .pdf | PyMuPDF로 텍스트 직접 추출 → Gemini 텍스트 분석 | PDFParser |
| PDF (스캔) | .pdf | 페이지별 이미지 렌더링 → Gemini 멀티모달 분석 (OCR) | PDFParser |
| Word | .docx | python-docx로 텍스트 추출 | DocxParser |
| Word (구버전) | .doc | olefile로 텍스트 추출 | DocParser |
| 한글 | .hwp, .hwpx | HWP 파서로 텍스트 추출 | HWPParser |
| 이미지 | .jpg, .jpeg, .png, .tiff, .tif, .bmp, .webp | PNG 변환 → Gemini 멀티모달 분석 | ImageParser |

### 3.3 스캔 PDF 감지 로직

일반 텍스트 PDF와 스캔 PDF를 자동으로 구분한다. 다음 조건 중 하나라도 해당하면 스캔 PDF로 판단:

| 조건 | 기준값 | 설명 |
|------|--------|------|
| 빈 페이지 존재 | 1페이지 이상 텍스트 없음 | 이미지만 있는 페이지가 존재 |
| 페이지당 평균 글자 수 | 50자 미만 | 텍스트가 매우 적음 |
| 의미 있는 문자 비율 | 30% 미만 | 한글/영문/숫자 비율이 낮음 (깨진 텍스트) |

**이미지 최적화 파이프라인:**
- DPI 자동 조절: 2.0 → 1.5 → 1.0 (페이지당 4MB 초과 시 단계적 축소)
- 최대 페이지 수: 20페이지
- 개별 이미지 최대: 4MB/페이지
- 총 이미지 크기 최대: 18MB (Gemini API 제한 준수)

### 3.4 분석 진행 표시

파일 업로드 후 3단계 프로그레스 바가 표시된다:

| 단계 | 표시 텍스트 | 설명 |
|------|-----------|------|
| 1단계 | "파일 업로드 중..." | XHR 기반 실시간 퍼센트 표시 (0~100%) |
| 2단계 | "AI가 계약서를 분석하고 있습니다..." | Gemini API 호출 중 (스피너 애니메이션) |
| 3단계 | "업무 목록 생성 중..." | 응답 파싱 및 데이터 구조화 |

- 각 단계마다 진행 바의 색상과 너비가 변경됨
- 업로드 단계는 XMLHttpRequest의 progress 이벤트로 실시간 퍼센트 표시
- 분석/생성 단계는 스피너 회전 애니메이션 (1초 주기)

### 3.5 AI 추출 정보

Gemini에게 전달하는 시스템 프롬프트를 통해 다음 정보를 구조화된 JSON으로 추출한다:

**계약 기본 정보:**
| 필드 | 설명 | 형식 |
|------|------|------|
| contract_name | 계약명/사업명 | 문자열 |
| company_name | 기업명 | 문자열 또는 null |
| contractor | 수급자 (을) | 문자열 또는 null |
| client | 발주처 (갑) | 문자열 또는 null |
| contract_date | 계약일 | YYYY-MM-DD 또는 원문 |
| contract_start_date | 착수일 | YYYY-MM-DD 또는 원문 |
| contract_end_date | 완수일 | YYYY-MM-DD 또는 원문 |
| total_duration_days | 총 사업 기간 | 숫자(일) 또는 null |
| contract_amount | 계약 금액 | 문자열 (원문 그대로) |
| payment_method | 지급 방식 | 문자열 또는 null |
| payment_due_date | 입금 예정일 | YYYY-MM-DD 또는 원문 또는 null |

**단계별 추진 일정 (schedules 배열):**
| 필드 | 설명 |
|------|------|
| phase | 단계명 (예: "1단계: 요구사항 분석") |
| schedule_type | 착수 / 설계 / 개발 / 테스트 / 납품 / 중간보고 / 최종보고 / 검수 / 인도 / 기타 |
| start_date | 시작일 (YYYY-MM-DD) |
| end_date | 종료일 (YYYY-MM-DD) |
| description | 단계 설명 |
| deliverables | 산출물 목록 (문자열 배열) |

**업무 목록 자동 생성 (task_list 배열):**
| 필드 | 설명 |
|------|------|
| task_id | 자동 부여 ID (정수) |
| task_name | 업무명 |
| phase | 해당 단계명 |
| due_date | 마감일 (YYYY-MM-DD) |
| priority | 긴급 / 높음 / 보통 / 낮음 |
| status | 초기값: "대기" |

### 3.6 분석 결과 화면

AI 분석이 완료되면 결과를 보여주는 화면으로 전환된다:

**좌측 영역 - 계약 기본 정보:**
- 계약명, 기업명, 수급자, 발주처 등 텍스트 필드
- 계약일, 착수일, 완수일 날짜 필드
- 계약 금액, 지급 방식 필드
- 모든 필드는 인라인 편집 가능 (클릭하여 수정)

**중앙 영역 - 추진 일정:**
- 단계별 카드 형태로 일정 표시
- 각 카드: 단계명, 시작일~종료일, 산출물 목록

**우측/하단 영역 - 업무 목록:**
- 테이블 형태로 업무 목록 표시
- 각 행: 업무명, 단계, 마감일, 우선순위, 상태
- 업무 추가/삭제 가능

**하단 버튼:**
- "계약 저장" 버튼 → 저장 후 축하 애니메이션 (confetti) 재생
- "다시 분석" 버튼 → 새 파일로 재시도

### 3.7 API 엔드포인트

| 메서드 | 경로 | 설명 | Rate Limit |

**요청:** Multipart form (file 필드)

**응답:**


**오류 응답:**
| 상황 | 상태 코드 | 메시지 |

| 분석 타임아웃 | 500 | "분석 시간이 초과되었습니다" |

## 4. 대시보드

### 4.1 화면 구성

대시보드는 로그인 후 기본 화면이며, 모든 계약의 업무 현황을 한눈에 보여준다.

**상단 - 통계 카드 (3개):**

| 카드 | 배경색 | 아이콘 | 표시 내용 |
|------|--------|--------|---------|
| 대기 | 노란색 (amber) | 시계 | 대기 상태 업무 수 |
| 진행중 | 파란색 (blue) | 화살표 | 진행중 업무 수 |
| 완료 | 초록색 (green) | 체크 | 완료 업무 수 |

- 각 카드를 클릭하면 해당 상태의 업무만 필터링
- 카드에 마우스 오버 시 약간 확대 효과 (scale)

**중앙 - 업무 목록:**

- **뷰 전환 토글**: 리스트 뷰 / 타임라인 뷰
- **필터 바**: 상태 필터 (전체/대기/진행중/완료) + 텍스트 검색 (300ms 디바운스)
- **검색 대상**: 업무명, 계약명, 단계명 (대소문자 무시)

### 4.2 리스트 뷰

업무를 카드 형태의 목록으로 표시한다. 각 업무 카드 구성:

| 영역 | 내용 |
|------|------|
| 좌측 상단 | 업무명 (굵은 글씨) |
| 우측 상단 | D-Day 배지 (마감일 기준) |
| 좌측 하단 | 계약명 (소속 계약 표시, 계약별 색상 구분) |
| 중앙 | 단계명 |
| 우측 하단 | 우선순위 배지 + 상태 변경 드롭다운 |

**D-Day 표시 규칙:**
| 조건 | 표시 | 색상 | 효과 |
|------|------|------|------|
| 기한 초과 | D+N | 빨간색 | - |
| 오늘 마감 | D-Day | 빨간색 | pulse 애니메이션 |
| 1~3일 남음 | D-1 ~ D-3 | 빨간색 배경 | - |
| 4~7일 남음 | D-4 ~ D-7 | 주황색 | - |
| 8일 이상 남음 | D-N | 회색 | - |
| 마감일 없음 | 표시 없음 | - | - |

**카드 확장 (클릭 시):**
- 카드를 클릭하면 확장되어 상세 정보 표시
- 상태 변경 드롭다운 (대기/진행중/완료)
- 담당자 지정 드롭다운 (팀 멤버 목록)
- 처리 내용 메모 입력 (textarea)
- 증빙 파일 업로드 영역 (20MB 제한)
- 댓글 섹션
- slideInDown 애니메이션으로 자연스럽게 펼침

**계약별 색상:**
- 각 계약에 고유 색상이 자동 배정됨
- 업무 카드 좌측에 색상 바로 표시
- 동일 계약의 업무는 같은 색상으로 시각적 그룹핑

### 4.3 타임라인 뷰

업무를 시간순 간트 차트 형태로 표시한다:
- 가로축: 날짜 (시작일~종료일)
- 세로축: 업무 목록
- 각 업무를 막대로 표시 (시작일부터 마감일까지)
- 오늘 날짜에 빨간 세로 기준선 표시
- 우선순위에 따라 막대 색상 구분

### 4.4 업무 상태 변경

업무 카드에서 상태를 변경할 수 있다:
1. 상태 드롭다운 클릭
2. 대기 / 진행중 / 완료 중 선택
3. 즉시 서버에 PATCH 요청 전송
4. 성공 시 카드 색상/배지 실시간 변경 + 통계 카드 갱신
5. 팀 멤버에게 알림 자동 발송

### 4.5 독립 업무 추가

대시보드에서 계약과 무관한 업무를 직접 추가할 수 있다:
1. "업무 추가" 버튼 클릭
2. 모달 창 표시: 업무명, 계약 선택 (선택사항), 마감일, 우선순위
3. 계약을 선택하면 해당 계약에 연결, 미선택 시 "미분류" 업무로 생성

## 5. 계약 관리

### 5.1 계약 저장

분석 결과 화면에서 "계약 저장" 클릭 시:
1. 계약명 중복 확인 (동일 사용자 범위)
2. 중복 시 오류: "동일한 이름의 계약서가 이미 존재합니다"
3. 저장 성공 시 축하 애니메이션 (confetti) 재생 (2.5초)
4. 대시보드로 자동 전환

**저장되는 데이터:**
| 필드 | 타입 | 설명 |
|------|------|------|
| contract_name | String | 계약명 |
| file_name | String | 원본 파일명 |
| company_name | String | 기업명 |
| contractor | String | 수급자 |
| client | String | 발주처 |
| contract_date | String | 계약일 |
| contract_start_date | String | 착수일 |
| contract_end_date | String | 완수일 |
| total_duration_days | Integer | 총 사업 기간 (일) |
| contract_amount | String | 계약 금액 |
| payment_method | String | 지급 방식 |
| payment_due_date | String | 입금 예정일 |
| schedules | JSON | 추진 일정 배열 |
| tasks | JSON | 업무 목록 배열 |
| milestones | JSON | 마일스톤 배열 |
| raw_text | TEXT | 계약서 원문 |

### 5.2 마이페이지 (내 계약 목록)

**화면 구성:**
- 상단: "내 계약서" 제목 + 연도/월 필터 드롭다운
- 본문: 계약 목록 (카드 형태, 최신순 정렬)

**각 계약 카드:**
| 영역 | 내용 |
|------|------|
| 제목 | 계약명 |
| 부제 | 기업명 |
| 날짜 | 착수일 ~ 완수일 |
| 금액 | 계약 금액 |
| 하단 버튼 | 상세보기 / 수정 / 삭제 |

**필터 동작:**
- 연도 드롭다운: 계약 생성 연도 기준 필터
- 월 드롭다운: 계약 생성 월 기준 필터
- 필터 변경 시 목록 즉시 갱신
- 페이지네이션: 20건 단위

**계약 삭제:**
1. 삭제 버튼 클릭
2. 커스텀 확인 다이얼로그 표시 (빨간색 위험 모드)
3. "정말 삭제하시겠습니까?" 확인 시 삭제
4. 연관 데이터 (댓글, 활동 로그) 함께 삭제

**계약 수정:**
1. 수정 버튼 클릭 → 분석 결과 화면을 편집 모드로 전환
2. 기본 정보, 일정, 업무 목록 수정 가능
3. "수정 저장" 버튼으로 변경사항 반영

### 5.3 데이터 내보내기

분석 결과 또는 계약 상세 화면에서 3가지 형식으로 내보내기 가능:

| 형식 | 버튼 | 내용 |
|------|------|------|
| CSV | "CSV 다운로드" | 업무 목록 (UTF-8 BOM, 한글 호환) |
| JSON | "JSON 다운로드" | 전체 분석 결과 (계약 정보 + 일정 + 업무) |
| Word | "Word 다운로드" | HTML 기반 .doc 형식 문서 |

| 보통 | 파란색 | 일반적인 업무 |
| 낮음 | 회색 | 여유 있는 처리 가능 |

- 대시보드 통계 실시간 갱신

**담당자 지정:**

**처리 내용 (메모):**
- 업무 카드 확장 시 textarea 표시

**증빙 파일 첨부:**

|------|------|
| 최대 크기 | 20MB |

| 업로드 | 파일 선택 버튼 클릭 |
| 다운로드 | 첨부 파일명 클릭 |
| 삭제 | X 버튼 클릭 (확인 다이얼로그) |

### 6.4 업무 추가

- 계약 상세 화면의 업무 목록에서 "업무 추가" 버튼
- 업무명, 단계, 마감일, 우선순위 입력

**독립 업무 추가:**
- 대시보드에서 "업무 추가" 버튼
- 계약 선택 드롭다운 (미선택 가능)
- 계약 선택 시 해당 계약에 연결

### 6.5 업무 삭제
1. 업무 카드의 삭제 아이콘 클릭
2. 커스텀 확인 다이얼로그 표시

## 7. 팀 관리

### 7.1 팀 개요

팀은 멤버 관리를 위한 독립 기능이다. 계약/업무와 직접 연동되지 않으며, 구성원 초대 및 역할 관리를 위한 용도이다.

### 7.2 팀 진입점

팀 기능에 접근하는 3가지 경로:

| 위치 | UI 요소 | 동작 |
|------|---------|------|
| 헤더 | 팀 드롭다운 메뉴 | 항상 표시, 팀 목록 및 "팀 만들기" 옵션 |
| 대시보드 | "팀 관리" 또는 "팀 만들기" 버튼 | 팀 선택 여부에 따라 다른 버튼 표시 |
| 모바일 하단 탭 | "팀" 탭 | 하단 내비게이션 바에 항상 표시 |

### 7.3 팀 생성

1. "팀 만들기" 버튼 클릭
2. 모달 창: 팀 이름 (필수), 설명 (선택)
3. 생성 완료 시 생성자가 자동으로 owner 역할 부여
4. 팀 관리 화면으로 전환

### 7.4 팀 관리 화면

**상단 영역:**
- 팀 이름 + 설명
- "팀 정보 수정" 버튼 (owner/admin만 표시)
- "팀 삭제" 버튼 (owner만 표시, 빨간색)

**멤버 목록 영역:**
| 열 | 내용 |
|----|------|
| 프로필 | 이메일 기반 아바타 (이니셜) |
| 이름 | 사용자 이름 |
| 이메일 | 사용자 이메일 |
| 역할 | 역할 배지 (owner/admin/member/viewer) |
| 액션 | 역할 변경 드롭다운 + 제거 버튼 |

**멤버 초대:**
1. "멤버 초대" 버튼 클릭
2. 이메일 입력 필드 표시
3. 이메일 입력 후 "초대" 버튼 클릭
4. 해당 이메일이 가입된 사용자인지 확인
5. 미가입 사용자: "등록되지 않은 사용자입니다" 오류
6. 이미 멤버: "이미 팀 멤버입니다" 오류
7. 초대 성공: member 역할로 추가, 초대 알림 발송

### 7.5 역할 및 권한

| 역할 | 팀 정보 수정 | 팀 삭제 | 멤버 초대 | 멤버 제거 | 역할 변경 |
|------|:----------:|:------:|:--------:|:--------:|:--------:|
| owner | O | O | O | O | O |
| admin | O | X | O | O | X |
| member | X | X | X | X | X |
| viewer | X | X | X | X | X |

**역할 변경 규칙:**
- owner만 다른 멤버의 역할을 변경할 수 있음
- owner 자신의 역할은 변경 불가
- 변경 가능한 역할: admin, member, viewer

**제약사항:**
- 마지막 owner는 삭제 불가 (최소 1명 유지 필수)
- owner 역할은 강등 불가

### 7.6 API 엔드포인트

| 메서드 | 경로 | 설명 | Rate Limit |
|--------|------|------|------------|
| POST | `/api/v1/teams` | 팀 생성 | 10회/분 |
| GET | `/api/v1/teams` | 내 팀 목록 | - |
| GET | `/api/v1/teams/{id}` | 팀 상세 + 멤버 목록 | - |
| PUT | `/api/v1/teams/{id}` | 팀 정보 수정 | 10회/분 |
| DELETE | `/api/v1/teams/{id}` | 팀 삭제 (owner만) | - |
| POST | `/api/v1/teams/{id}/members` | 멤버 초대 | 10회/분 |
| DELETE | `/api/v1/teams/{id}/members/{user_id}` | 멤버 제거 | - |
| PATCH | `/api/v1/teams/{id}/members/{user_id}/role` | 역할 변경 (owner만) | - |
| GET | `/api/v1/teams/{id}/permissions` | 내 팀 권한 조회 | - |

## 8. 댓글

### 8.1 댓글 위치

댓글은 두 곳에서 작성할 수 있다:
- **계약 댓글**: 계약 상세 화면의 댓글 섹션
- **업무 댓글**: 업무 카드 확장 시 하단 댓글 영역

### 8.2 댓글 작성

1. 댓글 입력 영역에 텍스트 입력 (최대 5,000자)
2. "등록" 버튼 클릭 또는 Enter 키
3. 댓글이 목록 하단에 추가됨
4. 시간순 정렬 (오래된 순 → 새로운 순)

**댓글 카드 구성:**
| 영역 | 내용 |
|------|------|
| 좌측 | 프로필 아바타 (이미지 또는 이니셜) |
| 상단 | 작성자 이름 + 작성 시간 (상대 시간) |
| 본문 | 댓글 내용 |
| 하단 | 수정/삭제 버튼 (본인 댓글) + 이모지 리액션 |

### 8.3 멘션 기능

댓글 작성 시 `@` 입력으로 사용자를 멘션할 수 있다:

1. 댓글 입력 중 `@` 입력
2. 자동완성 드롭다운 표시 (팀 멤버 목록)
3. 멤버 선택 시 `@이메일` 형식으로 자동 삽입
4. 댓글 등록 시 멘션된 사용자에게 알림 자동 발송

**멘션 인식 패턴:** `@(이메일주소)` 형식 (정규식으로 추출)

### 8.4 이모지 리액션

각 댓글에 이모지 리액션을 남길 수 있다:
- 제공 이모지: 👍 ❤️ 😄 🎉 🤔 👀
- 클릭으로 토글 (추가/제거)
- 리액션 수 배지 표시
- localStorage에 저장 (클라이언트 사이드)

### 8.5 댓글 수정/삭제

**수정:** 본인 댓글만 수정 가능 → 수정 아이콘 클릭 → 인라인 편집 모드 → 저장

**삭제:**
- 본인 댓글: 삭제 아이콘 클릭 → 확인 다이얼로그
- 팀 관리자 (owner/admin): 타인 댓글도 삭제 가능

### 8.6 상대 시간 표시

| 경과 시간 | 표시 |
|---------|------|
| 1분 미만 | "방금 전" |
| 1분~59분 | "N분 전" |
| 1시간~23시간 | "N시간 전" |
| 1일~6일 | "N일 전" |
| 7일 이상 | "YYYY-MM-DD" 형식 |

## 12. 보안

### 12.1 인증/세션 보안

| 항목 | 구현 방식 |
|------|---------|
| 비밀번호 저장 | bcrypt 해싱 (솔트 자동 생성) |

| 세션 쿠키 | HttpOnly, SameSite=Lax, 24시간 만료 |
| 타이밍 공격 방지 | 미등록 이메일에도 더미 해시 검증 수행 |
| 만료 세션 | 1시간 주기 자동 삭제 |
| 비밀번호 규칙 | 8자 이상, 영문 + 숫자 필수 |

### 12.2 HTTP 보안 헤더

```
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin
Strict-Transport-Security: max-age=31536000 (프로덕션)
```

### 12.3 입력 검증

| 필드 | 검증 규칙 | 오류 메시지 |
|------|---------|-----------|
| 이메일 | RFC 5321 (Pydantic) | "유효한 이메일 주소를 입력하세요" |
| 비밀번호 | 8자 이상, 영문+숫자 | "비밀번호는 8자 이상, 영문+숫자 필수" |
| 인증코드 | 6자리 숫자, 10분 유효 | "잘못된 인증코드입니다" |
| 계약명 | 1~500자, 범위 내 고유 | "동일한 이름의 계약서가 존재합니다" |
| 댓글 | 1~5,000자, 공백 불가 | "댓글은 5000자를 초과할 수 없습니다" |
| 파일 | 확장자 + Magic bytes | "파일이 손상되었거나 형식이 올바르지 않습니다" |
| 파일 크기 | 스트리밍 청크 단위 확인 | "파일 크기 초과" |
| task_id | 정규식 (TASK-\d{1-6}) | 경로 탐색 공격 차단 |
